#### VPN: How It Works ‚Äî 8 Key Outcomes

1. **VPN definition:** Encrypted tunnel to a VPN server; not full anonymity.
2. **Protection on the path:** Encrypts traffic against local network sniffing (public Wi-Fi).
3. **IP masking:** Sites see VPN server IP; reduces basic tracking.
4. **Protocol selection:** WireGuard/OpenVPN/IKEv2 trade speed vs compatibility vs stability.
5. **Leak prevention:** DNS leaks can expose visited domains; use leak protection.
6. **Kill switch:** Stops traffic if VPN drops to prevent real IP exposure.
7. **Split tunneling:** Useful for performance + local access, but can cause accidental leaks.
8. **Trust shift:** ISP sees less; VPN provider becomes a new trust point‚Äîchoose wisely.

**My applications:** public Wi-Fi safety, remote access, reducing IP exposure, and learning network security basics (DNS, protocols, routing).
